制作目的について

今回は目標を2つ設定しました。フルスクリーンで作品の紹介をする。アクセシビリティのボタンを表示するです。今回はJavascriptのライブラリをたくさん使用し、良いサイトを作ろうとも考えました。今回は新しいことに挑戦しつつ前から少しやりたかった、フルスクリーンのサイトと、最近気になっていたアクセシビリティに考慮したサイトの制作に特化しました。

制作したときの注意点

今回はアクセシビリティを組むこととjsのライブラリを使用することからまずは構成をどのようにするかから考えて作りました。特にアクセシビリティで操作する面のhtmlとcssの書き方は普段よりもっと考えて書くようにしました。正直今までここまでhtmlで悩んだのは初めてでした。変数の入れ子を作り文字、フォントサイズやpaddingやmarginはrem,em,vh。また必要に応じてcalc,maxなど指定方法などの書き方で作成しました。今回は画像が大きいということもあり画像の表示方法もサイズ調整に悩みました。またボタンのシステムを作成するのにJavascriptの量が多くなるので、関数で書きコード量を減らすように心がけました。

Javascript

ほとんどを関数化してなるべくコード数の削減とメンテナンス性の優れたコードを書くことを意識しました。chatgptはたくさん使いましたが、コードを書くうえで理解できないところは調べて知見を広げていくこともしました。またまた、jsのライブラリは英語のドキュメントとAIを駆使しながら作成しました。正直ライブラリはドキュメントがあるうえで更に、機能性が重充実しているのですごく良いと感じました。

TOPページ

ここでは、スライドとサムネイルを連携させて作成しました。スライドはswiperを使用しました。また、ダブルクリックでメインのURLに飛ぶようにしました。 公式ドキュメントをみつつ参考サイトを確認しながらなんとか作成しました。下記はswaipeのコードです。 


let mySwiper_main;
const myDelay = 5500;

//swiperのjs
const mySwiper_thumb = new Swiper(".mv .swiper-thumb", {
  slidesPerView: 4,
  spaceBetween: 15,
  direction: "vertical",
  roundLengths: true,
  speed: 1300,
  slideToClickedSlide: true,
  breakpoints: {
    601: {
      slidesPerView: 2,
      spaceBetween: 26,
    },
  },
  thumbs: {
    swiper: mySwiper_main,
  },
});

mySwiper_main = new Swiper(".mv .swiper-main", {
  direction: "vertical",
  loop: true,
  speed: 1200,
  loopAdditionalSlides: 1,
  allowTouchMove: false,
  autoplay: {
    delay: myDelay,
    disableOnInteraction: false,
    waitForTransition: false,
  },

  thumbs: {
    autoScrollOffset: 0.8,
    swiper: mySwiper_thumb,
  },
});

//ダブルクリックでメインのURL
document.addEventListener("DOMContentLoaded", () => {
  const thumbSlides = document.querySelectorAll(".swiper-thumb .swiper-slide");

  thumbSlides.forEach((slide, index) => {
    slide.addEventListener("dblclick", () => {
      const mainSlideLink = document
        .querySelector(\`.swiper-main .swiper-slide:nth-child(\${index + 1})\`)
        .getAttribute("href");

      window.location.href = mainSlideLink;
    });
  });
});

aboutページ

aboutページはなるべく伝えるべき面は先に伝え雰囲気を伝えた後、この業界で働きたいと思った経緯、目標、僕の生きがいの趣味について書いてあります。すごく長文ですが良ければ見ていただきたいです。動きがあり過ぎると面白くないなと感じ、gsapを使用して経歴のところに少しだけ動きをつけています。

アクセシビリティのボタン

アクセシビリティボタンについて解説していきたいと思います。

ボタンリスト

ボタンを紹介を左上から右に紹介したいと思います

  • 文字の大きさ変更
  • 文字の高さ変更
  • 文字の間隔を変更
  • 文字の色を選択し変更
  • 文字枠の色を変更
  • 背景色の変更
  • リンクハイライト
  • コントラスト
  • 彩度調整
  • アニメーションを停止
  • 画像非表示
  • 文字の位置を変更
  • フォント変更
  • カーソル変更

このようなボタンでサイト内の見え方を変更することができます。下記は細かな点な説明をしたいと思います。

色変更

色変更にはjsのpickrを使用しました。input=colorで実装は可能でしたが、jsのライブラリがあったので使って見たいと言う点とUIデザインが完成されていて機能の追加を簡単にできる点から使用することに至りました。デフォルトカラーを設定があったり、リセットをできるようにしています。

コントラストと彩度

cssのfilterで実現しているのですが、管理するうえでjsで更新する際に上書きされています点があり少し悩みました。cssのroot:を使うことで重ねがけを実現させています。

フォントの変更

フォントの変更は簡単でしたが、失読症の方のフォントを適用されるようにしました。日本語が”Sans Comic Suns”英語が”Open-Dyslexic”で表示されています。

カーソル

1クッリクで画像の適用。2クリックでカーソルの上下にカバーが掛けられてADHDの方に適用されたfilterが付きます。

関数化

関数化をなるべくするようにしました。


// テキストサイズの調整
adjustTextSize("textSize", "--font-size", [14, 17, 18, 19, 16]);

// 文字の高さの調整
adjustTextHeight("textHeight", "--font-height", [2.2, 2.4, 2.6, 2]);

// 文字の間隔
adjustTextSpace("textSpace", "--font-space", [0.1, 0.2, 0.3, 0.05]);

// コントラストフィルター
adjustContrast(
  "filter", // トリガー要素のIDを正確に指定
  "--contrast-filter",
  [
    "contrast(80%) grayscale(50%)",
    "contrast(140%)",
    "hue-rotate(180deg)",
    "",
  ],
  ["ライトコントラスト", "ハイコントラスト", "反転色", "コントラスト"]
);

// 彩度フィルター
adjustSaturate(
  "saturate",
  "--saturate-filter",
  ["saturate(50%)", "saturate(3)", "grayscale(100%)", ""],
  ["低彩度", "高彩度", "モノクロ", "彩度"]
);

// 文字の位置

);

adjustFont(

反省

デザイン

フルスクリーンでのデザインとなった為要素は最低限でしなければならなかったということもあり難しかった。画像の上に文字を配置している関係上コントラスト比の調整が難しい。アクセシビリティ特化のサイトにはならなかった。ただフルスクリーンの利点的に制作作品とメニューのダイナミックな表示がいい点。作品は大きく見えるのでギャラリー見たくいい感じる。また、ボタンのUIは参考を調べ作成したがいい感じに見やすいと思った。今回は色んな人にURLを送付してフィードバックをもらった。その中でいいものができたと思う。

コーディング

コーディング面はhtmlとcssと特に今回は気を使いどのようにするかだいたい決めてからコーディングをしました。結果アクセシビリティボタンの作成の成功とに至りました。ただ今回で悪い点もあったと思ったのはhtmlのアクセシビリティに考慮がかけている。デザインのは位置的にコントラスト比の考えに至れていないといった点です。正直アクセシビリティに特化サイトにするのならばそこに配慮すべき点だと思いました。コーディングと言いつつデザインですが正直もう少し構築を考えるべきだと思いました。

Javascript

今回のいい点は、Javascriptのライブラリを活用してたくさんの機能をつけたこと、swiperは前回使用したことがありその時はもっと簡単なスライドでしたが今回は複雑なスライドを作成できました。またアクセシビリティのボタンの機能を全て実現できたことです。こちらは関数に入れ実現することを努力しました。AIの使用は惜しみなくしましたが使用した場合は必ずコードの理解をしてから使用するようにしています。今後も色んなツール、ドキュメントを確認し開発をしていきたいと思います。
